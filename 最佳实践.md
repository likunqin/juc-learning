## 线程管理

### ✅ 推荐做法

1. **使用线程池而非手动创建线程**
   ```java
   // 推荐
   ExecutorService executor = Executors.newFixedThreadPool(10);

   // 不推荐
   for (int i = 0; i < 10; i++) {
       new Thread(() -> doWork()).start();
   }
   ```

2. **正确处理线程中断**
   ```java
   // 推荐
   public void run() {
       while (!Thread.currentThread().isInterrupted()) {
           try {
               doWork();
           } catch (InterruptedException e) {
               Thread.currentThread().interrupt(); // 恢复中断状态
               break;
           }
       }
   }
   ```

3. **使用有意义的线程名称**
   ```java
   Thread thread = new Thread(runnable, "订单处理线程-" + orderId);
   ```

### ❌ 避免做法

1. **不要手动创建大量线程**
   ```java
   // 错误：创建过多线程会导致资源耗尽
   for (int i = 0; i < 10000; i++) {
       new Thread(task).start();
   }
   ```

2. **不要忽视 InterruptedException**
   ```java
   // 错误：吞掉中断异常
   try {
       Thread.sleep(1000);
   } catch (InterruptedException e) {
       // 什么都不做
   }
   ```

## 锁的使用

### ✅ 推荐做法

1. **使用 try-finally 确保锁释放**
   ```java
   lock.lock();
   try {
       // 临界区代码
   } finally {
       lock.unlock();
   }
   ```

2. **尽量缩小锁的范围**
   ```java
   // 推荐
   synchronized (lock) {
       int result = compute();
   }
   // 更多不使用锁的代码

   // 不推荐
   synchronized (lock) {
       int result = compute();
       // 大量不使用锁的代码
   }
   ```

3. **考虑使用读写锁**
   ```java
   ReadWriteLock rwLock = new ReentrantReadWriteLock();

   // 读操作
   rwLock.readLock().lock();
   try {
       // 读操作
   } finally {
       rwLock.readLock().unlock();
   }

   // 写操作
   rwLock.writeLock().lock();
   try {
       // 写操作
   } finally {
       rwLock.writeLock().unlock();
   }
   ```

### ❌ 避免做法

1. **不要在锁内执行耗时操作**
   ```java
   // 错误：在锁内执行 IO 操作
   synchronized (lock) {
       networkCall(); // 阻塞操作
   }
   ```

2. **不要嵌套锁使用不同的顺序**
   ```java
   // 容易导致死锁
   synchronized (lock1) {
       synchronized (lock2) {
           // ...
       }
   }

   // 另一个地方
   synchronized (lock2) {
       synchronized (lock1) {
           // ...
       }
   }
   ```

3. **不要忘记释放锁**
   ```java
   // 错误：可能导致死锁
   lock.lock();
   doSomething(); // 如果抛出异常，锁不会被释放
   lock.unlock();
   ```

## 原子类

### ✅ 推荐做法

1. **简单计数使用 LongAdder**
   ```java
   // 推荐：高并发场景
   LongAdder counter = new LongAdder();
   counter.increment();

   // 也可以：AtomicLong
   AtomicLong atomicCounter = new AtomicLong();
   atomicCounter.incrementAndGet();
   ```

2. **复合操作使用 AtomicReference**
   ```java
   AtomicReference<User> userRef = new AtomicReference<>();
   userRef.updateAndGet(u -> new User(u.getId(), u.getName()));
   ```

### ❌ 避免做法

1. **不要用 volatile 代替 synchronized**
   ```java
   // 错误：复合操作不原子
   private volatile int counter;
   counter++; // 读-改-写，不原子
   ```

2. **不要在循环中使用 CAS 自旋
   ```java
   // 错误：自旋可能太长
   while (!atomic.compareAndSet(old, new)) {
       // 自旋等待
   }
   ```

## 并发集合

### ✅ 推荐做法

1. **选择合适的并发集合**
   ```java
   // Map
   ConcurrentHashMap<String, User> map = new ConcurrentHashMap<>();

   // List（读多写少）
   CopyOnWriteArrayList<String> list = new CopyOnWriteArrayList<>();

   // Queue
   BlockingQueue<Task> queue = new LinkedBlockingQueue<>();
   ```

2. **使用 ConcurrentHashMap 的原子方法**
   ```java
   // 推荐
   map.putIfAbsent(key, value);
   map.compute(key, (k, v) -> v == null ? 1 : v + 1);

   // 不推荐
   if (!map.containsKey(key)) {
       map.put(key, value); // 非原子操作
   }
   ```

### ❌ 避免做法

1. **不要使用 Collections.synchronizedList 等**
   ```java
   // 不推荐：性能较差
   List<String> list = Collections.synchronizedList(new ArrayList<>());

   // 推荐：使用专门的并发集合
   CopyOnWriteArrayList<String> list = new CopyOnWriteArrayList<>();
   ```

2. **不要在遍历时修改集合**
   ```java
   // 错误：会抛出 ConcurrentModificationException
   for (String s : list) {
       if (s.startsWith("a")) {
           list.remove(s);
       }
   }
   ```

## 线程池

### ✅ 推荐做法

1. **避免使用 Executors 工厂方法**
   ```java
   // 推荐：明确配置线程池
   ThreadPoolExecutor executor = new ThreadPoolExecutor(
       corePoolSize,
       maximumPoolSize,
       keepAliveTime,
       TimeUnit.SECONDS,
       new LinkedBlockingQueue<>(100),
       new ThreadFactoryBuilder().setNameFormat("worker-%d").build(),
       new ThreadPoolExecutor.CallerRunsPolicy()
   );

   // 不推荐：使用默认配置
   ExecutorService executor = Executors.newFixedThreadPool(10);
   ```

2. **正确设置线程池大小**
   ```java
   // CPU 密集型
   int cpuThreads = Runtime.getRuntime().availableProcessors() + 1;

   // IO 密集型
   int ioThreads = Runtime.getRuntime().availableProcessors() * 2;
   ```

3. **优雅关闭线程池**
   ```java
   executor.shutdown(); // 停止接受新任务
   try {
       if (!executor.awaitTermination(60, TimeUnit.SECONDS)) {
           executor.shutdownNow(); // 强制关闭
       }
   } catch (InterruptedException e) {
       executor.shutdownNow();
       Thread.currentThread().interrupt();
   }
   ```

### ❌ 避免做法

1. **不要使用无界队列**
   ```java
   // 错误：队列无界，可能导致 OOM
   new ThreadPoolExecutor(1, 1,
       0L, TimeUnit.MILLISECONDS,
       new LinkedBlockingQueue<Runnable>()); // 无界
   ```

2. **不要让线程任务无限运行**
   ```java
   // 错误：没有检查中断
   public void run() {
       while (true) { // 无限循环
           doWork();
       }
   }

   // 推荐
   public void run() {
       while (!Thread.currentThread().isInterrupted()) {
           doWork();
       }
   }
   ```

## 异步编程

### ✅ 推荐做法

1. **使用 CompletableFuture 组合操作**
   ```java
   CompletableFuture<String> userFuture = getUser(userId);
   CompletableFuture<String> orderFuture = getOrders(userId);

   // 组合两个 Future
   CompletableFuture<Result> resultFuture = userFuture.thenCombine(
       orderFuture,
       (user, orders) -> new Result(user, orders)
   );
   ```

2. **正确处理异常**
   ```java
   future
       .thenApply(this::process)
       .exceptionally(e -> {
           log.error("处理失败", e);
           return defaultValue;
       });
   ```

### ❌ 避免做法

1. **不要阻塞 Future**
   ```java
   // 错误：阻塞等待
   String result = future.get(); // 可能永远阻塞

   // 推荐：设置超时
   String result = future.get(5, TimeUnit.SECONDS);
   ```

2. **不要忽略异常**
   ```java
   // 错误：异常被忽略
   CompletableFuture.runAsync(() -> {
       throw new RuntimeException("error");
   });
   ```

## 性能优化

### ✅ 推荐做法

1. **使用适当的并发级别**
   ```java
   // 设置合理的并发级别
   ConcurrentHashMap<String, String> map =
       new ConcurrentHashMap<>(16, 0.75f, 16);
   ```

2. **批量操作**
   ```java
   // 批量转移，提高效率
   queue.drainTo(collection, 100);
   ```

3. **使用 ThreadLocal 缓存**
   ```java
   private static final ThreadLocal<SimpleDateFormat> dateFormat =
       ThreadLocal.withInitial(() -> new SimpleDateFormat("yyyy-MM-dd"));
   ```

1. **不要过早优化**
   ```java
   // 先确保正确性，再优化
   ```

2. **不要过度同步**
   ```java
   // 不要为了线程安全而过度同步
   ```

## 常见陷阱

### 1. 死锁

**原因**：
- 锁获取顺序不一致
- 资源循环等待

**预防**：
- 统一锁获取顺序
- 使用 tryLock 带超时
- 减少锁持有时间

### 2. 内存泄漏

**原因**：
- ThreadLocal 未清理
- 线程未关闭
- 资源未释放

**预防**：
- ThreadLocal.remove()
- 正确关闭线程池
- 使用 try-finally 释放资源

### 3. 性能问题

**原因**：
- 过度同步
- 锁竞争激烈
- 不必要的对象创建

**预防**：
- 缩小锁范围
- 使用并发集合
- 使用对象池

### 4. 竞态条件

**原因**：
- 复合操作不原子
- 检查再执行（check-then-act）

**预防**：
- 使用原子类
- 使用 synchronized
- 使用 CAS 操作
