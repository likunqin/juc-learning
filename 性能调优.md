## 性能分析

### 性能指标

1. **吞吐量**（Throughput）
   - 单位时间内完成的任务数
   - 衡量系统处理能力

2. **响应时间**（Response Time）
   - 从请求到响应的时间
   - 包括 P50、P95、P99

3. **延迟**（Latency）
   - 单个操作的等待时间

4. **并发度**（Concurrency）
   - 同时处理的请求数

### 性能分析工具

#### 1. JMH（Java Microbenchmark Harness）

专业的 Java 微基准测试框架。

**依赖**：
```xml
<dependency>
    <groupId>org.openjdk.jmh</groupId>
    <artifactId>jmh-core</artifactId>
    <version>1.36</version>
</dependency>
<dependency>
    <groupId>org.openjdk.jmh</groupId>
    <artifactId>jmh-generator-annprocess</artifactId>
    <version>1.36</version>
    <scope>provided</scope>
</dependency>
```

**示例**：
```java
@BenchmarkMode(Mode.AverageTime)
@OutputTimeUnit(TimeUnit.MILLISECONDS)
@Warmup(iterations = 3, time = 1)
@Measurement(iterations = 5, time = 1)
@Fork(1)
public class MyBenchmark {
    @Benchmark
    public void testMethod() {
        // 被测试的代码
    }
}
```

#### 2. Java Flight Recorder（JFR）

JDK 自带的性能分析工具。

**启动应用**：
```bash
java -XX:StartFlightRecording=duration=60s,filename=recording.jfr -jar app.jar
```

#### 3. VisualVM

可视化的性能分析工具。

```bash
jvisualvm
```

#### 4. Arthas

阿里巴巴开源的 Java 诊断工具。

```bash
java -jar arthas-boot.jar
```

**常用命令**：
- `thread`：查看线程信息
- `jad`：反编译类
- `watch`：观察方法调用
- `trace`：追踪方法调用链

## 锁优化

### 1. 减少锁的持有时间

**优化前**：
```java
synchronized (lock) {
    compute(); // 快速
    ioOperation(); // 慢速
    anotherCompute(); // 快速
}
```

**优化后**：
```java
int result1;
synchronized (lock) {
    result1 = compute();
}
ioOperation(); // 不在锁内
int result2;
synchronized (lock) {
    result2 = anotherCompute();
}
```

### 2. 使用读写锁

**适用场景**：读多写少

```java
ReadWriteLock rwLock = new ReentrantReadWriteLock();

// 读操作
rwLock.readLock().lock();
try {
    // 读操作
} finally {
    rwLock.readLock().unlock();
}

// 写操作
rwLock.writeLock().lock();
try {
    // 写操作
} finally {
    rwLock.writeLock().unlock();
}
```

### 3. 使用乐观锁

**适用场景**：冲突率低

```java
AtomicInteger atomic = new AtomicInteger(0);

// CAS 操作
int oldValue = atomic.get();
int newValue = oldValue + 1;
while (!atomic.compareAndSet(oldValue, newValue)) {
    oldValue = atomic.get();
    newValue = oldValue + 1;
}

// 或使用更简洁的方式
atomic.incrementAndGet();
```

### 4. 锁粗化

**适用场景**：连续获取同一个锁

```java
// 优化前
synchronized (lock) {
    op1();
}
synchronized (lock) {
    op2();
}
synchronized (lock) {
    op3();
}

// 优化后（JIT 会自动优化）
synchronized (lock) {
    op1();
    op2();
    op3();
}
```

### 5. 锁消除

JIT 会自动消除不必要的锁。

```java
// 优化前
public void method() {
    Object lock = new Object();
    synchronized (lock) {
        // lock 没有逃逸，可以消除
    }
}

// JIT 会优化为
public void method() {
    // 直接执行，无锁
}
```

## 线程池调优

### 1. 确定线程池大小

#### CPU 密集型任务

```java
int corePoolSize = Runtime.getRuntime().availableProcessors() + 1;
```

#### IO 密集型任务

```java
int corePoolSize = Runtime.getRuntime().availableProcessors() * 2;
```

#### 混合型任务

根据实际测试调整，从以下值开始：
```java
int corePoolSize = (int)(Runtime.getRuntime().availableProcessors() * 1.5);
```

### 2. 设置合理的队列大小

**有界队列**：
```java
BlockingQueue<Runnable> workQueue = new LinkedBlockingQueue<>(100);
```

**队列大小计算**：
```
队列大小 = (目标响应时间 - 任务平均处理时间) * 目标吞吐量
```

### 3. 选择合适的拒绝策略

**CallerRunsPolicy**：调用者执行（推荐）
```java
new ThreadPoolExecutor.CallerRunsPolicy()
```

**AbortPolicy**：抛出异常
```java
new ThreadPoolExecutor.AbortPolicy()
```

**DiscardPolicy**：静默丢弃
```java
new ThreadPoolExecutor.DiscardPolicy()
```

**DiscardOldestPolicy**：丢弃最老的任务
```java
new ThreadPoolExecutor.DiscardOldestPolicy()
```

### 4. 监控线程池状态

```java
class ThreadPoolMonitor {
    public void monitor(ThreadPoolExecutor executor) {
        System.out.println("活跃线程数: " + executor.getActiveCount());
        System.out.println("当前线程数: " + executor.getPoolSize());
        System.out.println("队列大小: " + executor.getQueue().size());
        System.out.println("已完成任务数: " + executor.getCompletedTaskCount());
        System.out.println("总任务数: " + executor.getTaskCount());
    }
}
```

### 5. 动态调整线程池

```java
// 增加核心线程数
executor.setCorePoolSize(newCorePoolSize);

// 增加最大线程数
executor.setMaximumPoolSize(newMaximumPoolSize);
```

## 并发集合优化

### 1. 选择合适的并发集合

| 场景 | 推荐集合 |
|------|----------|
| 高并发 Map | ConcurrentHashMap |
| 读多写少 List | CopyOnWriteArrayList |
| 高并发 Queue | LinkedBlockingQueue |
| 有序操作 | ConcurrentSkipListMap |
| 延迟操作 | DelayQueue |

### 2. 设置合理的初始容量

```java
// 避免频繁扩容
ConcurrentHashMap<String, String> map =
    new ConcurrentHashMap<>(initialCapacity);
```

**初始容量计算**：
```
initialCapacity = 预期元素数 / 负载因子
```

### 3. 设置合理的并发级别（ConcurrentHashMap JDK 7）

```java
ConcurrentHashMap<String, String> map =
    new ConcurrentHashMap<>(16, 0.75f, concurrencyLevel);
```

**并发级别**：
- 默认值：16
- 建议值：CPU 核心数 * 2

### 4. 使用批量操作

```java
// 批量转移
queue.drainTo(collection, 100);

// 批量添加
map.putAll(otherMap);
```

## 内存优化

### 1. 避免内存泄漏

**ThreadLocal 使用**：
```java
try {
    threadLocal.set(value);
    // 使用
} finally {
    threadLocal.remove(); // 必须清理
}
```

**线程池使用**：
```java
executor.shutdown();
executor.awaitTermination(60, TimeUnit.SECONDS);
```

### 2. 使用对象池

```java
// 使用 Apache Commons Pool
ObjectPool<MyObject> pool = new GenericObjectPool<>(
    new MyObjectFactory());

try {
    MyObject obj = pool.borrowObject();
    try {
        // 使用对象
    } finally {
        pool.returnObject(obj);
    }
}
```

### 3. 使用基本类型

```java
// 优先使用基本类型
int[] array = new int[1000]; // 而非 Integer[]
```

### 4. 避免自动装箱

```java
// 错误
List<Integer> list = new ArrayList<>();
for (int i = 0; i < 1000; i++) {
    list.add(i); // 自动装箱
}

// 正确
IntStream.range(0, 1000).forEach(i -> {
    // 使用基本类型流
});
```

## JVM 调优

### 1. JIT 编译优化

**启用 C2 编译器**：
```bash
-XX:+TieredCompilation
```

**编译阈值**：
```bash
-XX:CompileThreshold=10000
```

### 2. GC 调优

**G1 GC**（推荐）：
```bash
-XX:+UseG1GC
-XX:MaxGCPauseMillis=200
-XX:G1HeapRegionSize=16m
```

**CMS GC**：
```bash
-XX:+UseConcMarkSweepGC
-XX:+UseParNewGC
```

**Parallel GC**：
```bash
-XX:+UseParallelGC
-XX:ParallelGCThreads=4
```

### 3. 内存配置

**堆内存**：
```bash
-Xms4g -Xmx4g
```

**新生代**：
```bash
-Xmn2g
```

**永久代/元空间**：
```bash
-XX:MetaspaceSize=256m -XX:MaxMetaspaceSize=512m
```

### 4. 其他优化参数

**偏向锁**：
```bash
-XX:+UseBiasedLocking
```

**自适应自旋**：
```bash
-XX:+UseSpinning
-XX:PreBlockSpin=10
```

**大页内存**：
```bash
-XX:+UseLargePages
```

## 性能测试策略

### 1. 基准测试

使用 JMH 进行微基准测试。

### 2. 压力测试

模拟高并发场景。

### 3. 稳定性测试

长时间运行，检测内存泄漏。

### 4. 对比测试

对比不同实现的性能。

## 性能调优流程

1. **建立基线**
   - 记录当前性能指标
   - 确定优化目标

2. **分析瓶颈**
   - 使用性能分析工具
   - 定位热点代码

3. **实施优化**
   - 针对瓶颈进行优化
   - 小步迭代

4. **验证效果**
   - 对比优化前后性能
   - 确保功能正确性

5. **持续监控**
   - 生产环境监控
   - 定期回顾优化效果

## 注意事项

1. **不要过早优化**
   - 先确保正确性
   - 基于性能数据优化

2. **不要过度优化**
   - 考虑投入产出比
   - 关注整体性能

3. **考虑维护性**
   - 代码清晰很重要
   - 避免过度复杂化

4. **持续关注**
   - 性能优化是一个持续过程
   - 随着需求变化调整策略
