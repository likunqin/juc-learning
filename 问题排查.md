## 死锁排查

### 症状

- 程序卡住不动
- CPU 使用率正常或为 0
- 线程处于 BLOCKED 或 WAITING 状态

### 排查方法

#### 1. 使用 ThreadMXBean

```java
ThreadMXBean threadMXBean = ManagementFactory.getThreadMXBean();
long[] deadlockedThreads = threadMXBean.findDeadlockedThreads();

if (deadlockedThreads != null) {
    for (long threadId : deadlockedThreads) {
        ThreadInfo threadInfo = threadMXBean.getThreadInfo(threadId);
        System.out.println("死锁线程: " + threadInfo.getThreadName());
        System.out.println("阻塞信息: " + Arrays.toString(threadInfo.getLockedMonitors()));
    }
}
```

#### 2. 使用 jstack

```bash
# 获取进程 ID
jps

# 生成线程转储
jstack <pid> > thread_dump.txt

# 查找死锁
jstack <pid> | grep -A 10 "Found one Java-level deadlock"
```

#### 3. 使用 jcmd

```bash
jcmd <pid> Thread.print

# 或生成文件
jcmd <pid> Thread.print_to_file -file thread_dump.txt
```

#### 4. 使用 VisualVM

1. 启动 VisualVM
2. 连接到应用
3. 查看 Threads 标签页
4. 检测死锁（点击 "Detect Deadlock"）

### 预防措施

1. **统一锁顺序**
   - 所有线程按相同顺序获取锁
   - 使用对象的 hashCode 确定顺序

2. **使用 tryLock 超时**
   ```java
   if (lock.tryLock(1, TimeUnit.SECONDS)) {
       try {
           // 临界区
       } finally {
           lock.unlock();
       }
   }
   ```

3. **减少锁持有时间**
   - 只在必要时持有锁
   - 避免在锁内执行耗时操作

## 内存泄漏排查

### 症状

- 内存持续增长
- 频繁 Full GC
- 最终 OutOfMemoryError

### 排查方法

#### 1. 使用 jmap

```bash
# 查看堆内存使用
jmap -heap <pid>

# 导出堆转储
jmap -dump:format=b,file=heap_dump.hprof <pid>
```

#### 2. 使用 VisualVM 分析堆转储

1. File -> Load -> 选择堆转储文件
2. 查看 "Classes" 标签页
3. 查找占用内存最多的类
4. 分析对象引用关系

#### 3. 使用 MAT（Memory Analyzer Tool）

1. 打开堆转储文件
2. 使用 "Leak Suspects Report" 自动分析
3. 使用 "Dominator Tree" 查找大对象
4. 使用 "Histogram" 按类查看对象数量

### 常见内存泄漏场景

#### 1. ThreadLocal 泄漏

**问题代码**：
```java
ThreadLocal<byte[]> threadLocal = new ThreadLocal<>();
threadLocal.set(new byte[1024 * 1024]); // 1MB
// 忘记调用 remove()
```

**正确代码**：
```java
try {
    threadLocal.set(new byte[1024 * 1024]);
} finally {
    threadLocal.remove();
}
```

#### 2. 静态集合泄漏

**问题代码**：
```java
private static Map<String, Object> cache = new HashMap<>();

public void put(String key, Object value) {
    cache.put(key, value); // 永不清理
}
```

**正确代码**：
```java
// 使用 WeakHashMap
private static Map<String, Object> cache = new WeakHashMap<>();

// 或使用 Guava Cache
private static Cache<String, Object> cache = CacheBuilder.newBuilder()
    .expireAfterWrite(10, TimeUnit.MINUTES)
    .build();
```

#### 3. 监听器泄漏

**问题代码**：
```java
public class EventSource {
    private List<EventListener> listeners = new ArrayList<>();

    public void addListener(EventListener listener) {
        listeners.add(listener); // 永不移除
    }
}
```

**正确代码**：
```java
public class EventSource {
    private List<EventListener> listeners = new CopyOnWriteArrayList<>();

    public void addListener(EventListener listener) {
        listeners.add(listener);
    }

    public void removeListener(EventListener listener) {
        listeners.remove(listener);
    }
}
```

## 性能问题排查

### 症状

- 响应时间变慢
- CPU 使用率高
- 吞吐量下降

### 排查方法

#### 1. 使用 JFR

```bash
# 启动时录制
java -XX:StartFlightRecording=duration=60s,filename=recording.jfr -jar app.jar

# 或运行时录制
jcmd <pid> JFR.start duration=60s filename=recording.jfr
```

使用 JDK Mission Control 分析 JFR 文件。

#### 2. 使用 Arthas

```bash
# 追踪方法执行时间
trace com.example.Service method

# 查看线程 CPU 使用
thread -all

# 查看方法调用统计
profiler start
profiler getSamples
```

#### 3. 使用 VisualVM Profiler

1. 启动 VisualVM
2. 连接到应用
3. 点击 "Profiler" 标签页
4. 开始/停止 Profiling
5. 查看热点方法

### 常见性能问题

#### 1. 锁竞争激烈

**症状**：
- 大量线程在等待锁
- CPU 使用率不高但响应慢

**排查**：
```java
ThreadMXBean threadMXBean = ManagementFactory.getThreadMXBean();
long[] threadIds = threadMXBean.getAllThreadIds();

for (long threadId : threadIds) {
    ThreadInfo info = threadMXBean.getThreadInfo(threadId);
    if (info.getThreadState() == Thread.State.BLOCKED) {
        System.out.println("BLOCKED 线程: " + info.getThreadName());
        System.out.println("等待的锁: " + info.getLockName());
    }
}
```

**解决方案**：
- 减少锁粒度
- 使用读写锁
- 使用并发集合
- 考虑乐观锁（CAS）

#### 2. 上下文切换频繁

**症状**：
- CPU 使用率高但吞吐量低
- 大量线程状态切换

**排查**：
```bash
# 使用 vmstat 查看 context switches
vmstat 1

# 使用 jstack 查看线程状态
jstack <pid> | grep "java.lang.Thread.State"
```

**解决方案**：
- 减少线程数量
- 减少锁等待
- 使用协程/虚拟线程

## 线程问题排查

### 症状

- 线程池拒绝任务
- 线程长时间阻塞
- 线程泄漏

### 排查方法

#### 1. 使用 jstack 分析线程状态

```bash
jstack <pid> | grep -A 5 "java.lang.Thread.State"
```

常见线程状态：
- **RUNNABLE**：正在运行
- **BLOCKED**：等待锁
- **WAITING**：等待其他线程通知
- **TIMED_WAITING**：带超时的等待

#### 2. 分析线程转储

1. 获取线程转储：
   ```bash
   jstack <pid> > thread_dump.txt
   ```

2. 查找可疑线程：
   - BLOCKED 线程（死锁）
   - WAITING 线程（可能卡住）
   - 大量相同状态的线程

### 常见线程问题

#### 1. 线程泄漏

**问题代码**：
```java
// 不正确关闭线程池
ExecutorService executor = Executors.newCachedThreadPool();
executor.submit(task); // 提交后不关闭
```

**正确代码**：
```java
ExecutorService executor = Executors.newCachedThreadPool();
try {
    executor.submit(task);
} finally {
    executor.shutdown();
    executor.awaitTermination(60, TimeUnit.SECONDS);
}
```

#### 2. 任务卡住

**问题代码**：
```java
// 没有超时控制
future.get(); // 可能永远阻塞
```

**正确代码**：
```java
// 设置超时
future.get(5, TimeUnit.SECONDS);
```

#### 3. 忽略中断

**问题代码**：
```java
while (true) { // 不检查中断
    doWork();
}
```

**正确代码**：
```java
while (!Thread.currentThread().isInterrupted()) {
    doWork();
}
```

## 常见错误

### 1. ConcurrentModificationException

**问题代码**：
```java
for (String s : list) {
    if (condition) {
        list.remove(s); // 抛出异常
    }
}
```

**正确代码**：
```java
// 使用迭代器
Iterator<String> it = list.iterator();
while (it.hasNext()) {
    String s = it.next();
    if (condition) {
        it.remove();
    }
}

// 或使用并发集合
CopyOnWriteArrayList<String> list = new CopyOnWriteArrayList<>();
```

### 2. IllegalMonitorStateException

**问题代码**：
```java
Object lock = new Object();
lock.notify(); // 错误：不在 synchronized 块中
```

**正确代码**：
```java
synchronized (lock) {
    lock.notify();
}
```

### 3. RejectedExecutionException

**问题代码**：
```java
ThreadPoolExecutor executor = new ThreadPoolExecutor(
    1, 1,
    0, TimeUnit.SECONDS,
    new LinkedBlockingQueue<>(1) // 有界队列
);

for (int i = 0; i < 100; i++) {
    executor.submit(task); // 队列满时抛出异常
}
```

**正确代码**：
```java
// 1. 使用无界队列（谨慎）
ThreadPoolExecutor executor = new ThreadPoolExecutor(
    1, 1,
    0, TimeUnit.SECONDS,
    new LinkedBlockingQueue<>()
);

// 2. 设置拒绝策略
executor.setRejectedExecutionHandler(new ThreadPoolExecutor.CallerRunsPolicy());

// 3. 使用 Future 捕获异常
Future<?> future = executor.submit(task);
try {
    future.get();
} catch (RejectedExecutionException e) {
    // 处理拒绝
}
```

### 4. NullPointerException（并发场景）

**问题代码**：
```java
class Shared {
    private volatile Object value;

    public Object getValue() {
        return value.toString(); // value 可能为 null
    }
}
```

**正确代码**：
```java
class Shared {
    private volatile Object value;

    public Object getValue() {
        Object v = value;
        return v != null ? v.toString() : defaultValue;
    }
}
```

## 排查工具推荐

1. **JDK 工具**
   - jps：查看 Java 进程
   - jstack：线程转储
   - jmap：堆转储
   - jstat：统计信息
   - jcmd：多功能工具

2. **VisualVM**
   - JDK 自带
   - 监控和 Profiling

3. **JFR & JMC**
   - 高性能分析
   - 低开销

4. **MAT**
   - 内存分析
   - 泄漏检测

5. **Arthas**
   - 在线诊断
   - 热更新

6. **JProfiler**
   - 商业工具
   - 功能强大

## 排查流程

1. **收集信息**
   - 线程转储
   - 堆转储
   - GC 日志
   - 应用日志

2. **分析问题**
   - 确定问题类型
   - 定位问题位置

3. **验证假设**
   - 代码审查
   - 单元测试
   - 压力测试

4. **实施修复**
   - 小步修改
   - 验证效果

5. **预防措施**
   - 监控告警
   - 代码规范
   - 定期检查

## 最佳实践

1. **保持日志完整**
   - 记录关键操作
   - 记录异常信息

2. **监控关键指标**
   - 线程池状态
   - 内存使用
   - GC 情况

3. **建立基准**
   - 正常性能水平
   - 异常判断依据

4. **定期检查**
   - 线程转储
   - 堆使用情况
   - 性能指标
